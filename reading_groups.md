### Related Work (Sec. 5) on object-centric learning:
* Key works like "Slot Attention" (Locatello et al., 2020), "Sound of Pixels" (Zhao et al., 2018), "Objects That Sound" (Arandjelović et al., 2017), "SSV2A" (Guo et al., 2024), "AVObject" (Li et al., 2025).
* A. [Slot Attention] The key difference is that Slot Attention sets the slots first and then trains the mapping of objects to these slots, whereas OOP learns the spatial, frequency, and temporal relationships of the mixture audio first and then separates the objects through an object splitter.
* The main difference between these works and the proposed OOP is that, unlike OOP’s core characteristic, these works do not process the separated features in parallel by permuting them along the batch dimension for each task (localization, separation, retrieval, and audio generation). In OOP, features are separated by the object splitter and processed independently in the batch dimension, fundamentally preventing interference from other object features.
* [Sound of Pixels] is a network that separates audio occurring at specific pixels on a pixel-level basis. It extracts video features through a video analysis network and audio features through an audio analysis network and then performs source separation in an audio synthesis network. This approach is similar to conventional audio source separation networks, which extract information for source separation and ultimately perform the separation.
* [Object that Sounds] proposes a network that fuses video and audio features for cross-modal retrieval, enabling semantic object localization. Similar to the previous Sound of Pixels, it utilizes object-centric learning. If OOP were applied to this work, features could be extracted through AVEnet, and the object splitter would separate each audio feature, allowing parallel retrieval of each sound source at the batch level.
* [SSV2A, AVObject] also employ object-centric learning in the multi-modal domain, similar to the previous two papers.
* In conclusion, the key difference between object-centric learning and OOP lies in whether objects are explicitly separated and then processed in parallel through a parameter-shared decoder by passing them as batches.
